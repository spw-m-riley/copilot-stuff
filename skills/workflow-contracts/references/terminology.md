# Terminology

Use these terms consistently across planning, review, and implementation surfaces.

## Reference skill

A reusable skill whose main value is discoverability, conventions, lookup-heavy guidance, or examples. It helps an agent find or decide something quickly without forcing a heavy execution playbook.

## Task skill

A reusable skill that drives a concrete multi-step workflow. It should make inputs, outputs, guardrails, and validation explicit.

## Handoff artifact

A durable structured markdown artifact that carries enough context between phases without depending on the full chat history.

## Review outcome

A structured reviewer response with an explicit status plus evidence and a next action. In this workflow, the status is `approve`, `revise`, or `blocked`.
