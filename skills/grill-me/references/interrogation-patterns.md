# Interrogation patterns

Lightweight guidance for running an effective grilling session. For the full playbook including domain-doc maintenance patterns, see `skills/grill-with-docs/references/session-playbook.md`.

## Question sequencing

- Resolve foundational concepts before dependent decisions.
- Walk one branch of the decision tree at a time — finish it before starting the next.
- Provide your recommended answer with every question so the user has something concrete to react to.

## When to explore instead of ask

- If the question is "how does X currently work?" — explore the codebase.
- If the question is "should X work this way?" — ask the user.
- When in doubt, check the code first and present what you found.

## When to stop

The session is complete when every branch of the decision tree has been resolved and the user confirms shared understanding.
